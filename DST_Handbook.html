Serializing an Item

The team will indicate in the body of an email when to serialize item(s). Once serialized, the UIN Type and UIN Label fields should be visible on the Locations screen. If not, use View → Columns to add them.

Serializing a Single Item

  1. Look up the RIN at the Transaction Level in RMS v16.
  2. Navigate to More Actions → Locations.
  3. Scroll down and select More Actions → Location Attributes.
  4. Click the Green Plus Sign (+). In the Add Locations box, use the dropdown to select All Stores → click OK.
  5. Scroll to the bottom right, select the required UIN Type and UIN Label options from the dropdowns → click Save and Close.
  6. Save and Close out of the item.

Serializing Multiple Items (Mass Change)

  1. Create an Item List in RMS v16 (can be Parent + Transactions, or just Transactions).
  2. Search for Item List in RMS v16.
  3. Click Actions → Mass Change → Item/Locations.
  4. Click the Green Plus Sign (+). Select All Stores → click OK.
  5. For mass change, the Primary Supplier of the items must be entered in both the Primary Supplier and Primary Country fields.
  6. Select the required UIN Type and UIN Label options → click Save and Close.

Creating a v16 Item from an Existing v10 Item

When an item exists in RMS v10 but not in RMS v16, follow the appropriate process below based on the item's level structure.

Level 2 Item (2-Level)

  1. Populate the Item Induction template as usual.
  2. Go to the Item Master tab. Enter the RIN from the Transaction Item column in the Item column for Level 2 rows.
  3. Scroll to the Original Reference Number column and copy the system-generated Level 2 item number.
  4. Press CTRL+H → open Find & Replace. Click Options, set Within to Workbook. Replace the system-generated number with the v10 RIN. Confirm a large number of replacements.
  5. Delete the item RIN in the Level 2 row of the Original Reference Number column (leave blank).
  6. Repeat for any additional RINs.
  7. Confirm UDAs have populated on the Item_Free_Form_UDAs tab.
  8. Save as .ods and induct as usual.

Level 3 Item (3-Level)

  1. Populate the Item Induction template as usual.
  2. Go to the Item Master tab. Enter the Parent RIN (from the first tab) in the Item column for Level 1 rows. Enter the Transaction RIN in the Item column for Level 2 rows.
  3. Use workbook-wide Find & Replace (CTRL+H → Options → Workbook) to replace the Level 1 Item Family ID with the original Parent RIN. Confirm replacements.
  4. Perform another workbook-wide Find & Replace to replace the system-generated Level 2 number with the transaction RIN from v10. Confirm replacements.
  5. Delete the RINs in both the Level 1 and Level 2 rows of the Original Reference Number column (leave blank).
  6. When complete: the Item column on the Item Master tab should be fully filled; the Original Reference Number column should be entirely blank.
  7. Confirm UDAs have populated on the Item_Free_Form_UDAs tab.
  8. Save as .ods and induct as usual.

Level 3 Item — Parent Already Exists in v16

Rare scenario, typically occurs with Softlines when rebuilding a specific transaction on an already-converted parent RIN.

  1. Change Action (Column A) to Add transaction or GTIN.
  2. Copy the Transaction RIN(s) and paste off to the side for later use.
  3. Delete Transaction RIN(s) from Transaction Item (Column I).
  4. Populate the template normally.
  5. Go to the Item Master tab. Enter the saved Transaction RIN in the Item column (Column B) for the Level 2 rows (cell above the corresponding GTIN).

⚠

  2. Select the .ods file, assign a process name → Upload.
ℹ

Franchise Costing Location & RPM Pricing Impact

Accurate RPM retail pricing depends on consistent supplier and costing data across all locations within a price zone. Two common setup issues can negatively affect RPM calculations and margin outcomes.

⚠ Inconsistent Primary Supplier in a Zone

All locations within a price zone must share the same primary supplier. When a location has a different supplier, item costs for that location differ from the rest of the zone.

RPM averages cost across all locations (including warehouses) when calculating retail price. Inconsistent suppliers skew this average, producing a retail price that is too high or too low.

⚠ Franchise Locations Missing Costing Location

Franchise locations must have a valid costing location assigned at the item/location level. Without one, RPM may use incomplete, default, or incorrect cost data.

Missing costing locations distort the cost average used by RPM, further affecting the suggested retail price across the zone.

Combined Impact on Margin

⚠
When either issue exists in a price zone, RPM still includes all locations when averaging cost. Incorrect or missing costs skew the calculated average — and the price guide is applied to a distorted cost base.
  • Overstated or understated margins at the item and zone level
  • Unexpected margin erosion or inflated pricing
  • Inconsistent margins across locations within the same price zone
  • Reduced confidence in margin reporting and pricing decisions
  • Increased need for manual price overrides and corrections

Affected Price Strategy departments: 962 – Beverages 982 – Spirits 983 – Wine 984 – Beer 962 – Completed

3rd Party Gift Cards (TPGC)

⚠
Do NOT create 3rd Party Gift Cards in XAdmin. TPGC items are created in RMS as non-merchandise and configured via MNT files only.

Process Overview

  1. Create the 3rd Party Gift Card item in RMS as Non-Merchandise.
  2. Create and upload the spreadsheet for STS.
  3. Create an Incident Ticket for the STS Team to add new items to the lookup table.
  4. Submit TPGC RINs and Fee RINs to the ORUP Team.
  5. Using the MNT Creator file, create the required MNT files for the card type (see matrix below).
  6. Upload into XAdmin and Deploy.

MNT Files Required by Card Type

The number and type of .mnt files needed vary by gift card type. Refer to the XAdmin Gift Card MNT File Matrix in the Non-Merch SOP (Section 9.1.2) for the complete breakdown.

Key MNT file tabs used for TPGC creation include: GCITMProperties, TPGAttachFee, TPGC_UPC, TPGNonMerchandise, and ItemPriceAction.

Updating Template Checker Tables

Whenever a new Supplier, UDA, Brand, or Size is created in RMS v16, the Item Induction Template Checker tables must be updated. Run the appropriate SQL query and use the results to refresh the checker.

UDA Query

SELECT U1.UDA_ID, U1.UDA_DESC, U2.UDA_VALUE, U2.UDA_VALUE_DESC FROM rms.uda u1, rms.UDA_VALUES u2 WHERE U1.UDA_ID = U2.UDA_ID ----AND u1.uda_id = 5261 ORDER BY U1.UDA_ID, U2.UDA_VALUE;

Supplier Query

SELECT supplier, duns_number, supplier_parent, sup_name, sup_status, create_datetime FROM rms.sups ORDER BY supplier;

Brand Query

SELECT * FROM rms.brand ORDER BY BRAND_NAME;

Size Query

SELECT DGD.DIFF_GROUP_ID, DGH.DIFF_GROUP_DESC, DI.DIFF_ID, DGD.CREATE_DATETIME, DGD.CREATE_ID, DGD.LAST_UPDATE_DATETIME, DGD.LAST_UPDATE_ID FROM RMS.DIFF_GROUP_DETAIL dgd, RMS.DIFF_GROUP_HEAD dgh, RMS.DIFF_IDS di WHERE DI.DIFF_ID = DGD.DIFF_ID AND DGD.DIFF_GROUP_ID = DGH.DIFF_GROUP_ID --AND DI.DIFF_ID = '0' AND DGD.DIFF_GROUP_ID <> 'COLOR' AND NOT (DI.DIFF_ID = '0' AND DGD.DIFF_GROUP_ID = '262S') --AND DI.CREATE_DATETIME > sysdate - 30 ORDER BY DGD.DIFF_GROUP_ID, DGH.DIFF_GROUP_DESC, DI.DIFF_ID;

Mandatory UDA Query

SELECT DISTINCT ud.dept, c.class, c.class_name, s.subclass, s.sub_name, ud.uda_id, u.uda_desc, ud.required_ind, ud.hierarchy_value, u.CREATE_ID, u.CREATE_DATETIME, uv.UDA_VALUE, uv.UDA_VALUE_DESC, uv.CREATE_ID, uv.CREATE_DATETIME FROM rms.uda_item_defaults ud, rms.uda u, rms.uda_values uv, rms.class c, rms.subclass s, rms.deps d WHERE ud.uda_id = u.uda_id AND U.UDA_ID = uv.UDA_ID AND d.dept = ud.dept AND d.dept = c.dept AND c.class = s.class AND d.dept = s.dept AND ud.class = c.class;
